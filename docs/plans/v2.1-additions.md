## High-Value Additions (sorted by bang-for-buck)

### üî• Tier 1 ‚Äî Everyone Needs These

#### Array (from es-toolkit / radash)

| Utility | What | Why |
|---------|------|-----|
| `countBy(arr, fn)` | Count occurrences by key | Very common for analytics/dashboards |
| `minBy(arr, fn)` / `maxBy(arr, fn)` | Find min/max by function | Every data table needs this |
| `orderBy(arr, criteria, orders)` | Multi-key sort with asc/desc | `sortBy` only does single key |
| `union(a, b)` | Unique values from both | Complement to `intersection`/`difference` |
| `xor(a, b)` | Symmetric difference | Elements in either but not both |
| `sample(arr)` / `sampleSize(arr, n)` | Random element(s) | Common for games/randomization |
| `takeWhile(arr, fn)` / `dropWhile(arr, fn)` | Conditional take/drop | Much more useful than fixed `take`/`drop` |
| `without(arr, ...values)` | Remove specific values | Cleaner than `filter` |
| `zipObject(keys, values)` | Create object from key/value arrays | Complement to `zip` |

#### Object (from es-toolkit / radash)

| Utility | What | Why |
|---------|------|-----|
| `findKey(obj, fn)` | Find key by predicate | Very common pattern |
| `flattenObject(obj, delimiter?)` | `{ a: { b: 1 } }` ‚Üí `{ 'a.b': 1 }` | Forms, APIs, analytics |
| `toCamelCaseKeys(obj)` / `toSnakeCaseKeys(obj)` | Convert all object keys | API ‚Üî frontend data transform (#1 asked for) |

#### String (from es-toolkit / radash)

| Utility | What | Why |
|---------|------|-----|
| `pascalCase(str)` | Convert to PascalCase | Component naming convention |
| `startCase(str)` | 'hello world' ‚Üí 'Hello World' | UI display formatting |
| `template(str, data, regex?)` | `'Hello, {{name}}'` ‚Üí `'Hello, Ray'` | Simple string interpolation |
| `pad(str, length, chars?)` | Pad both sides | Missing alongside `padStart`/`padEnd` |
| `deburr(str)` | Remove diacritics: 'caf√©' ‚Üí 'cafe' | Search normalization, URLs |
| `escapeRegExp(str)` | Escape regex special chars | Every search input needs this |

#### Function (from es-toolkit)

| Utility | What | Why |
|---------|------|-----|
| `negate(fn)` | Returns `!fn(...)` | `arr.filter(negate(isNil))` ‚Äî very ergonomic |

#### Promise / Async (from es-toolkit / radash)

| Utility | What | Why |
|---------|------|-----|
| `attempt(fn)` / `attemptAsync(fn)` | `[error, result]` tuple pattern | Go-style error handling, replaces try/catch |
| `parallel(limit, items, fn)` | Concurrent async with limit | Rate-limited API calls, batch processing |
| `invariant(condition, msg)` | Assert condition or throw | Type narrowing + runtime assertion |

#### Predicate (from es-toolkit)

| Utility | What | Why |
|---------|------|-----|
| `isPromise(v)` | Promise type guard | Common in async utilities |
| `isError(v)` | Error type guard | Error handling patterns |
| `isRegExp(v)` | RegExp type guard | Completes the guard set |
| `isMap(v)` / `isSet(v)` | Map/Set type guards | Completes the guard set |
| `isPrimitive(v)` | Primitive check | Useful in serialization |

## Recommended Implementation Priority

### Phase A ‚Äî Quick Wins, Max Impact (~2 hours)
These are small, tested, zero-dependency utilities everyone uses:

1. `countBy`, `minBy`, `maxBy`, `union`, `xor`, `without` (array)
2. `negate` (function)
3. `attempt`, `attemptAsync` (promise)
4. `invariant` (util)
5. `pascalCase`, `startCase`, `escapeRegExp`, `slugify` (string)
6. `findKey`, `flattenObject` (object)
7. `isPromise`, `isError`, `isPrimitive` (predicate)

### Phase B ‚Äî React Hooks for Real Apps (~3 hours)
Hooks from Tier 2 that 90% of apps need:

1. `useEventListener` ‚Äî foundation for many other hooks
2. `useInterval` / `useTimeout` ‚Äî declarative timers
3. `useWindowSize` ‚Äî replaces `useScreenSize`
4. `useHover` ‚Äî every interactive UI
5. `useDocumentTitle` ‚Äî every SPA
6. `useDisclosure` ‚Äî modals, drawers, sidebars
7. `useScrollLock` ‚Äî modal overlays
8. `useNetwork` ‚Äî offline handling
9. `useCounter` ‚Äî numeric inputs, quantities
10. `useUnmount` ‚Äî cleanup patterns
11. `useHotkeys` ‚Äî keyboard shortcuts

### Phase C ‚Äî UI/UX Color & Format (~2 hours)
Move beyond basic utils into UI-specific territory:

1. Color: `rgbaToHex`, `hexToHsl`, `lighten`, `darken`, `getContrastRatio`
2. Format: `formatBytes`, `formatDuration`, `relativeTime`, `pluralize`
3. String: `slugify`, `maskString`, `template`, `deburr`
4. Object: `toCamelCaseKeys`, `toSnakeCaseKeys`

### Phase D ‚Äî Advanced Async (~1 hour)
1. `parallel(limit, items, fn)` ‚Äî rate-limited concurrency
2. Move `attempt`/`attemptAsync`/`invariant` to new `util/` module

## What NOT to Add (and why)

| Skip | Reason |
|------|--------|
| `useFetch` | Use TanStack Query / SWR instead |
| `useGeolocation` / `useBattery` | Too niche, easy to write inline |
| `useAudio` / `useVideo` | Media handling is app-specific |
| Full date library | Use date-fns or dayjs |
| `Mutex` / `Semaphore` | Too advanced for a utility lib |
| `useSpring` / `useTween` | Use framer-motion |
| `isBuffer` / `isTypedArray` | Too niche, Node.js specific |
| `curryRight` / `partialRight` | Diminishing returns |
| `flatMapDeep` / `flattenDeep` | `arr.flat(Infinity)` exists |
